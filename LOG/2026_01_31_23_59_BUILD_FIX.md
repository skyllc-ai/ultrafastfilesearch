# Build Fix: Rust Nightly Compatibility Issue

**Date**: 2026-01-31 23:59  
**Issue**: Build failure due to Polars incompatibility with recent nightly Rust versions  
**Status**: ✅ RESOLVED

## Problem

The build was failing with two different errors depending on the nightly version:

### Error 1: Latest Nightly (2026-01-30)
```
error[E0432]: unresolved imports `std::simd::LaneCount`, `std::simd::SupportedLaneCount`
 --> polars-arrow/src/bitmap/bitmask.rs:2:17
```
**Cause**: Breaking changes in `std::simd` API in recent nightly versions

### Error 2: Nightly 2026-01-15
```
thread 'rustc' panicked at compiler/rustc_resolve/src/imports.rs:379:13:
assertion failed: !deep_decl.is_glob_import()
```
**Cause**: Compiler bug in mid-January 2026 nightlies when compiling Polars

### Error 3: Stable (1.93.0)
```
error[E0554]: `#![feature]` may not be used on the stable release channel
 --> foldhash-0.2.0/src/lib.rs:115:34
```
**Cause**: Dependencies (foldhash, Polars) require nightly features

## Solution

Pinned to **nightly-2025-12-15** which is known to work with the current Polars git dependency.

### Changes Made

**File**: `rust-toolchain.toml`
```toml
[toolchain]
# Using nightly-2025-12-15 which is known to work with Polars git dependency
# Recent nightlies (2026-01-*) have compiler panics and SIMD breaking changes
# TODO: Update to latest nightly once Polars is compatible
channel = "nightly-2025-12-15"
```
